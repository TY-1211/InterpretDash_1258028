  理論對照：

     計算結果之 AUC 可能意味著數據中存在重要的特徵交互作用，而 EBM 自動偵測並加入成對交互項（Pairwise interactions）的功能發揮了作用，進而提升了準確度。

## 其他
- 比較資料不經過分割及測試後的AUC數值和探討原因
  1. 過擬合的幻象
     作弊效應：如果不分割資料，模型等於是在背答案。模型在評估階段看到的數據正是它在訓練階段學習過的數據。
     過度擬合細節：EBM 雖然是玻璃盒模型，但它仍使用梯度提升與袋裝法等現代技術來學習特徵函數。
     如果沒有測試集，模型會學習到數據中的隨機雜訊而非通用規律，導致 AUC 異常膨脹。
  2. 缺乏泛化能力驗證
     論文的基準：論文中的 AUC 數據是基於模型對「未見過數據」的預測能力 。
     這是衡量模型在現實世界中是否有效的唯一標準。
     評估偏差：沒有分割資料，衡量的是訓練準確度而非泛化準確度。     
  3. EBM 的學習特性
     高容量特徵函數：EBM 為了達到與隨機森林或 XGBoost 相當的準確度，其特徵函數非常靈活 。
     階梯函數的細節：如果資料沒分割，EBM 可能會為了擬合訓練集中的每一個離群點而產生極其複雜的階梯，這會讓訓練 AUC 極高，但在實際應用時表現會崩潰。
  
- 探討實質數據和模型預測差異的原因
  1. 數據分佈與偽陰性的權衡
     在論文討論的 AUC 評估中，模型是在追求整體預測的準確率。
     模型傾向：如果數據集中「健康的人」遠多於「患病的人」，模型為了獲得更高的整體 AUC 或準確率，可能會傾向於將模糊地帶的案例預測為「健康」。
     閾值設定：預測結果通常是基於一個機率值。如果一個患者的特徵表現不夠典型，模型算出的患病機率可能是 0.45，此時系統就會判定為「沒有心臟病」，造成預測失敗。
  2. 特徵的侷限性（遺漏關鍵指標）
     從 EBM 的角度來看，模型是根據輸入的特徵來計算得分。
     隱性因子：如果某位患者的發病主因是「遺傳基因」或「短期極度壓力」，但資料集中只包含「生理量測數值」，模型就無法捕捉到這些關鍵資訊。
     特徵貢獻度不足：在 Figure 2 的右側個體預測圖中，可以看到每個特徵的加權。如果患者在大部分指標都很正常，即便有一兩個指標略高，加總後的總分可能仍不足以跨過患病的門檻。
  3. 資料切分與過擬合問題
     若沒有正確進行 訓練與測試集分割，會導致模型產生過擬合的行為：
     泛化能力喪失：模型可能過度學習了訓練集中的特定雜訊。當面對一個特徵組合稍微不同的新病人時，模型會因為缺乏「泛化能力」而無法給出正確判斷，導致實際有病卻預測錯誤。
     資料洩漏幻象：沒有分割資料會讓您誤以為模型很準，但實際上模型只是在「背答案」，遇到真實案例時就會失效。
 4. 統計上的類別失衡
    心臟病預測常面臨樣本不平衡的問題。如果訓練資料中「有病」的案例太少，模型學到的「有病特徵」就不夠全面。當出現一種模型沒見過的發病型態時，它就會預設為最常見的類別——即「健康」。

- Q1：三個資料集的 EBM AUC 差異，你認為原因是什麼？
     我認為資料品質是影響 AUC 差異相當重要的一大因素。如果某個資料集中含有大量標註錯誤或缺失值，模型計算出的結果都會受雜訊干擾。另外，特徵與目標變數間缺乏強相關性也是干擾 AUC 因素之一。從資料集可直接查看，若呈現異常的離群值通常是導致 AUC 偏低的原因。 
     之前提及過若資料集比例極端會使模型不穩定，比例過度失衡會導致模型追求整體準確率而將大部分機率預測為負例，直接造成 AUC 下降，因模型偏差失去區分的罕見正例能力。
     特徵與樣本數影響模型的學習容量，若特徵數遠大於樣本數，EBM 的函數會變得過複雜，即使訓練 AUC 很高，但是測試集表現會崩潰。
